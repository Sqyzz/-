# 前后端代码健壮性与逻辑完整性测试用例

## 1. 房间人数限制与生命周期

| 编号 | 测试点                 | 测试方式                                      | 预期结果                     |
| ---- | ---------------------- | --------------------------------------------- | ---------------------------- |
| S-01 | 并发加入房间           | 同时发起 10 个 join 请求                      | 最多只有 5 人成功            |
| S-02 | Redis Lua 加入原子性   | 压测 join_room 接口                           | 不出现 6 人房间              |
| S-03 | 重复 join              | 同一 socket 重复加入                          | 不重复计入人数               |
| S-04 | 同一 playerId 重复加入 | 复用 playerId 请求加入                        | 幂等处理，不产生重复玩家     |
| S-05 | 离开房间幂等           | 连续发送 leave_room                           | 只处理一次，不报错           |
| S-06 | 房间状态机合法性       | waiting/countdown/playing/finished 多阶段切换 | 不允许非法跳转               |
| S-07 | 满员中断 30 秒倒计时   | 第 5 人加入时检查定时器                       | 30 秒 timer 被取消           |
| S-08 | 多 timer 泄漏          | 多次创建销毁房间                              | 无残留倒计时或 game loop     |
| S-09 | 空房间清理             | 玩家全部退出                                  | Redis 房间状态和定时器被释放 |
| S-10 | 游戏结束资源释放       | 60 秒结束                                     | 停止金币生成，清理临时锁     |

## 2. 金币抢占并发逻辑

| 编号 | 测试点           | 测试方式                              | 预期结果             |
| ---- | ---------------- | ------------------------------------- | -------------------- |
| S-11 | 同金币并发抢占   | 100 个请求同时 claim 同一 coinId      | 只有 1 个成功        |
| S-12 | SETNX 原子性     | 检查 Redis `coin:{id}:claimed`      | 只记录一个 playerId  |
| S-13 | 计分原子性       | 并发抢成功后检查分数                  | 只有赢家 +1          |
| S-14 | 失败请求不加分   | loser 请求返回失败                    | 分数不变             |
| S-15 | 重复 claim 幂等  | 同一玩家重复 claim 同一金币           | 最多加 1 分          |
| S-16 | 过期金币 claim   | coin TTL 到期后请求                   | 返回失败             |
| S-17 | 非当前金币 claim | 请求历史 coinId                       | 返回失败             |
| S-18 | 伪造 coinId      | 请求不存在 coinId                     | 返回失败             |
| S-19 | 伪造 roomId      | player 不属于 roomId                  | 返回失败             |
| S-20 | 伪造 playerId    | socket 绑定身份与请求 playerId 不一致 | 返回未授权或非法请求 |

## 3. 分数与事件一致性

| 编号 | 测试点                | 测试方式                              | 预期结果                                        |
| ---- | --------------------- | ------------------------------------- | ----------------------------------------------- |
| S-21 | 分数只由服务端修改    | 前端提交 score 字段                   | 服务端忽略                                      |
| S-22 | 广播顺序              | 抢金币后监听事件                      | 先 coin_claimed，再 score_updated，或按约定顺序 |
| S-23 | 最终积分一致性        | 游戏结束后查 Redis/DB/客户端          | 三方分数一致                                    |
| S-24 | 高并发多金币          | 多金币、多玩家压测                    | 无重复加分、无丢分                              |
| S-25 | 游戏结束瞬间点击      | 倒计时归零同时 claim                  | 以服务端状态为准，不越界加分                    |
| S-26 | 事件落库完整性        | 检查 coin_events                      | spawn、claim、winner、时间戳完整                |
| S-27 | 断线玩家抢金币        | 玩家离线后继续发送请求                | 请求失败                                        |
| S-28 | 非 playing 状态抢金币 | waiting/countdown/finished 阶段 claim | 全部拒绝                                        |

## 4. WebSocket 稳定性

| 编号 | 测试点      | 测试方式               | 预期结果                               |
| ---- | ----------- | ---------------------- | -------------------------------------- |
| S-29 | 心跳正常    | 客户端定期 heartbeat   | 玩家保持在线                           |
| S-30 | 心跳丢失    | 停止 heartbeat         | 超时后清理玩家                         |
| S-31 | 异常断开    | kill 浏览器进程        | 后端 eventually 清理连接               |
| S-32 | 重复连接    | 同一玩家多 socket 登录 | 按设计踢旧连接或绑定最新连接           |

| S-34 | Socket 重连 | 网络断开后重连         | 状态重新同步                           |
| S-35 | 消息乱序    | 人为延迟部分消息       | 客户端按 serverVersion/serverTime 修正 |
| S-36 | 大量空消息  | 发送非法事件名         | 服务端不崩溃                           |
