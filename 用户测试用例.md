# 用户侧功能完整测试用例

## 1. 房间大厅与加入流程

| 编号 | 测试点           | 操作步骤                   | 预期结果                                        |
| ---- | ---------------- | -------------------------- | ----------------------------------------------- |
| U-01 | 正常加入房间     | 输入房间号、昵称，点击加入 | 加入成功，进入等待房间                          |
| U-02 | 首个玩家加入     | 第一个玩家加入空房间       | 开启 30 秒准备倒计时                            |
| U-03 | 多人加入广播     | 第二个玩家加入同一房间     | 房间内所有玩家收到“新玩家加入”提示            |
| U-04 | 房间满员立即开局 | 5 名玩家在 30 秒内加入     | 中断 30 秒倒计时，进入 3 秒倒计时               |
| U-05 | 未满员自动开局   | 30 秒结束时房间人数 < 5    | 进入 3 秒倒计时并开始游戏                       |
| U-06 | 超员加入拒绝     | 第 6 名玩家尝试加入        | 接口拒绝，前端提示房间已满                      |
| U-07 | 昵称为空         | 昵称为空点击加入           | 前端阻止或后端返回参数错误                      |
| U-08 | 房间号为空       | 房间号为空点击加入         | 前端阻止或后端返回参数错误                      |
| U-09 | 重复昵称加入     | 同房间内使用相同昵称加入   | 按设计处理：允许但分配不同 playerId，或明确拒绝 |
| U-10 | 玩家主动退出     | 等待阶段点击退出           | 玩家离开房间，其他玩家收到退出广播              |

## 2. 游戏开始与倒计时

| 编号 | 测试点           | 操作步骤                 | 预期结果                       |
| ---- | ---------------- | ------------------------ | ------------------------------ |
| U-11 | 3 秒开局倒计时   | 房间进入开局阶段         | 所有客户端显示 3、2、1         |
| U-12 | 游戏 60 秒倒计时 | 游戏正式开始             | 顶部显示 60 秒倒计时并逐秒减少 |
| U-13 | 所有玩家同步开始 | 多客户端观察开始时间     | 各客户端基本同步进入游戏       |
| U-14 | 游戏结束         | 倒计时归零               | 停止刷金币，展示最终积分榜     |
| U-15 | 结束后禁止点击   | 游戏结束后点击金币或空格 | 不再计分，接口返回游戏已结束   |

## 3. 金币刷新与展示

| 编号 | 测试点           | 操作步骤               | 预期结果                                |
| ---- | ---------------- | ---------------------- | --------------------------------------- |
| U-16 | 金币随机刷新     | 游戏中等待金币出现     | 金币出现在 5x5 网格中的随机格子         |
| U-17 | 坐标合法性       | 多次观察金币位置       | 金币坐标始终在 0~4 或 1~5 的合法范围内 |
| U-18 | 多客户端同步显示 | 多个玩家同时观察       | 所有人看到同一个金币位置                |
| U-19 | 金币唯一 ID      | 连续刷新多个金币       | 每个金币有唯一 coinId                   |
| U-20 | 金币被抢后消失   | 任一玩家抢到金币       | 所有客户端该金币消失                    |
| U-21 | 新金币刷新       | 一个金币被抢后继续游戏 | 后续按随机间隔继续刷新金币              |

## 4. 抢金币与计分

| 编号 | 测试点             | 操作步骤                 | 预期结果                         |
| ---- | ------------------ | ------------------------ | -------------------------------- |
| U-22 | 单人正常抢到金币   | 玩家点击金币             | 服务端确认成功，该玩家 +1        |
| U-23 | 积分榜更新         | 玩家抢到金币后观察积分榜 | 所有客户端积分榜同步更新         |
| U-24 | 未点中金币         | 点击无金币格子           | 不加分，不影响金币状态           |
| U-25 | 同一金币重复点击   | 玩家连续点击同一个金币   | 前端只发送一次或极少量请求       |
| U-26 | 抢到后本地动画     | 点击金币后立即观察 UI    | 先播放点击动画，但分数不立即确定 |
| U-27 | 抢失败回滚         | 模拟该金币已被别人抢走   | 本地乐观动画结束，分数不增加     |
| U-28 | 分数以服务端为准   | 前端本地篡改分数         | 服务端广播后覆盖为真实分数       |
| U-29 | 多人同时抢同一金币 | 两个玩家几乎同时点击     | 只有一个玩家加 1 分              |
| U-30 | 抢旧金币           | 使用旧 coinId 请求抢金币 | 请求失败，不加分                 |

## 5. 玩家退出、断线与重连

| 编号 | 测试点         | 操作步骤               | 预期结果                           |
| ---- | -------------- | ---------------------- | ---------------------------------- |
| U-31 | 等待阶段退出   | 玩家在准备倒计时退出   | 房间人数减少并广播                 |
| U-32 | 游戏中退出     | 玩家游戏中关闭页面     | 其他玩家收到离线或退出状态         |
| U-33 | 房主退出       | 首个进入房间的玩家退出 | 游戏不应异常中断，除非设计为房主制 |
| U-34 | 所有人退出     | 房间内所有玩家退出     | 房间资源被清理                     |
| U-35 | 浏览器异常关闭 | 直接关闭浏览器窗口     | 后端心跳超时后清理玩家             |
| U-36 | 网络短暂断开   | 断网后恢复连接         | 按设计恢复房间状态或重新加入       |
| U-37 | 重连后积分恢复 | 游戏中断线重连         | 分数与当前游戏状态以服务端为准     |
